Helm User Manual

Comprehensive guide to the executive dashboard for the DockLabs suite.

FAQ

Overview

Helm is the executive dashboard for the DockLabs suite. It rolls up real-time metrics, action items, and alerts from every peer product into a single view, and hosts a lightweight project-management surface for cross-cutting work the peer products don't already own.

Cross-fleet dashboards

See real-time metrics from every DockLabs product in one executive view.

Awaiting Me

Items across the suite where you are the gating dependency.

Projects & tasks

Standard, FOIA, and Capital Improvement Plan projects with collaborators, notes, and attachments.

Compliance & audit

Statutory FOIA clocks, CIP federal-fund posture, full status history, and ops canaries.

Roles

Helm-specific roles are issued via Keel SSO and surfaced in your sidebar profile.

RoleCapability
Admin Full access. Manage projects, collaborators, archives, public visibility, FOIA tolling, GovQA push, and Project Online imports.
Director Read all dashboards and projects. Create projects, claim, collaborate, transition, comment, and attach.
Viewer Read-only across the dashboard and projects assigned to or shared with you.

Suite-level system_admin inherits Admin behavior on Helm.

Getting Started

Signing in
1 From any DockLabs product, click Helm in the fleet switcher, or visit https://helm.docklabs.ai/.
2 Click Sign in with DockLabs (the suite OIDC button).
3 You'll land on /dashboard/.
If you're already signed in to another DockLabs product, the redirect is seamless — no second login form.
What you'll see first
  • Today tab — your personal inbox: My Work, Awaiting Me, and Alerts.
  • Across the suite tab — fleet-wide metrics and drill-downs.

The active tab is sticky via the ?tab=today|suite query string, so reloads land you where you left off.

Dashboard — Today Tab

The Today tab is your personal stand-up view. Three columns:

1. My Work

A deadline rail of every open task assigned to you, grouped:

  • Overdue — past due, not done.
  • Today — due today.
  • This week — due in the next 7 days.
  • Upcoming — anything later.

The same predicate powers /tasks/ — the two surfaces never drift apart.

2. Awaiting Me

Cross-suite inbox. For each peer product where you have a role, Helm queries /api/v1/helm-feed/inbox/ and lists items where you are the gating dependency:

PeerWhat appears here
ManifestActive signing steps where you are the signer and the packet is in progress.
HarborOpen review assignments and processing assignments.
AdmiraltyFOIA requests assigned to you in an open status, statutory deadline shown as the due date.
PurserSubmissions you can review (your program, not yet reviewed).
BountyNew high-relevance opportunity matches for you.
BeaconKeep-in-touch reminders due in 3 days; pending company approvals you own.
LookoutTracked bills you're driving (researching, drafting testimony, collaborating).
YeomanSpeaking invitations assigned, principaled, or delegated to you.

If a peer hasn't enabled the per-user endpoint yet, Helm falls back to a coarse aggregate count and shows a ~N badge with a tooltip. Per-user-per-peer cache is 60s; fetches run in parallel (8 workers max).

3. Alerts

  • Local notifications — anything from Helm's own Notification table.
  • Aggregated unread notifications — unread counts surfaced by each peer's inbox payload.
  • Ops canaries (staff only) — pulled live from /api/v1/metrics/.

Dashboard — Across the Suite

The executive view. Renders:

  • Period bar — pick a window (current period, last period, custom).
  • Fleet metric grid — one card per product, primary KPI plus a sparkline. Click expand to load a drill-down panel via htmx.
  • Fleet-aggregate action queue — same items as Awaiting Me, rolled up across users for an org-wide view.
  • Fleet-aggregate alerts — peer alerts plus ops canaries.
  • Watch list — pinned products / metrics you've bookmarked.

Drill-down panels

Each per-product drill-down shows the full metric card, open action items, active alerts, and the data freshness timestamp (with a stale-data warning if the snapshot is older than the cache window). Drill-downs are read-only — to act on an item, click through into the peer product.

Projects

/tasks/projects/ — Helm's container for any work that has more than one task and more than one person. Each project carries a kind (Standard, FOIA, or CIP), a status, a lead (set via Claim), collaborators, tasks, notes, attachments, full status history, and a public ID for transparency views.

Lifecycle

Helm follows the DockLabs Project Lifecycle Standard:

claim → invite collaborators → diligence (notes + attachments) → stage progression → optional handoff to Manifest for signing → signed-doc roundtrip → optional downstream export

active ⇄ on_hold
active/on_hold → completed/cancelled
completed/cancelled → archived (terminal)
archived → active/completed/cancelled (unarchive restores prior state)

Pause and Cancel require a comment. Archive does not. Unarchive restores the project to whatever terminal status it had before (recorded in previous_terminal_status).

Creating a project

1Click Projects → New project.
2Pick a kind (Standard, FOIA, or CIP). FOIA and CIP unlock kind-specific fields.
3Set name, description, color, and target end date.
4Save, then click Claim to make yourself the LEAD, and Invite collaborators to add the team.

Tasks

Tasks live inside a project. Each carries a title, description, status, priority, assignee, due date, manual position (sort order), and full status history.

Status changes fire task_status_changed notifications to the task assignee and project collaborators.

Due-date notifications

A daily cron (helm-notify-due-tasks, 09:00 UTC) fires:

  • task_due_soon — 24 hours before due_date.
  • task_overdue — once when the task crosses due_date.

Both are idempotent (24-hour cooldown stamped on the task), so re-running the job won't spam.

Collaboration

Per-project roles

RoleWhat they can do
LEADDrive the project. Transition status, archive/unarchive, manage collaborators, toggle public visibility, run AI summary, push to GovQA / Manifest.
CONTRIBUTORCreate and update tasks, add notes, attach files, comment.
REVIEWERRead everything; comment on notes and attachments.
OBSERVERRead-only access — useful for executive visibility without blast radius.

Only LEAD / CONTRIBUTOR / REVIEWER / OBSERVER are valid. Custom role names are not supported (suite-wide convention).

Notes & attachments

  • Notes — internal staff-only by default. Mark a note "external" to expose it on the public transparency view.
  • Attachments — drag-and-drop uploads. Each carries a source (manual upload, manifest_signed, etc.) and a visibility flag. Files attached as manifest_signed are produced automatically by the Manifest signing roundtrip.

FOIA Projects

A project with kind = FOIA unlocks statutory-clock fields and a tolling control bar.

Statutory clock

FieldMeaning
JurisdictionConnecticut (CGS §1-206 — 4 business days to acknowledge) is the default. Federal (5 USC 552 — 20 business days) is also supported.
Received atThe date the request was received. Triggers the clock.
Statutory deadlineComputed deadline. Recomputes when received_at, jurisdiction, or tolling changes. Indexed; visible as a countdown badge on the project card and Awaiting Me column.
Tolled at / Tolled untilPause window. The deadline shifts forward by the tolled duration.

Tolling

LEAD users can pause and resume the clock:

  • Toll — record the start of a tolling period (e.g. waiting on a fee agreement, requester clarification). Surface a label and end date on submission.
  • Untoll — resume the clock. The deadline recomputes, advancing forward by the tolled duration.

Every toll/untoll is logged in status history.

Granicus GovQA push

If GRANICUS_GOVQA_URL and GRANICUS_GOVQA_API_KEY are set, LEAD users see a Push to GovQA button that creates / updates the matching record in Granicus GovQA. The button is hidden when the integration isn't configured — there are no silent no-ops.

Capital Improvement Plan (CIP) Projects

A project with kind = CIP unlocks fund sources and federal compliance flags.

Fund sources

A structured list of {source, amount_cents, label} entries. Sources include:

  • ARPA — American Rescue Plan Act
  • IIJA — Infrastructure Investment and Jobs Act
  • IRA — Inflation Reduction Act
  • BEAD — Broadband Equity, Access, and Deployment
  • SLCGP — State and Local Cybersecurity Grant
  • CDBG — Community Development Block Grant
  • General Obligation Bond / Revenue Bond
  • State Match / Local Match / General Fund

Totals per project are visible on the public transparency page when public visibility is enabled.

Federal compliance flags

  • Davis-Bacon — federal construction prevailing-wage requirements apply.
  • Build America, Buy America (BABA) — domestic procurement applies.
  • NEPA — federal environmental review required.
  • Environmental review — state or local environmental review required.

These are advisory flags. They document audit posture; they don't enforce blocking rules.

Public Transparency

Each project has a public_id (UUID) and a public_visibility flag (default private). LEAD-only toggle.

When public, the project is reachable at /p/<public_id>/ to anyone, no authentication required, and renders:

  • Project name, status, and target dates.
  • Task completion percentage.
  • Fund sources and totals (CIP only).
  • The FOIA statutory deadline countdown (FOIA only).
The public view does not expose notes, attachments, collaborators, comments, or any PII. It is a transparency primitive, not a sharing primitive.

Calendar

/tasks/calendar/ — FullCalendar-mounted view of project target end dates, task due dates, and FOIA statutory deadlines.

iCal subscribe

/tasks/calendar.ics — login-required iCal feed of the same items, so you can subscribe from Outlook / Apple Calendar / Google Calendar. The feed is per-user; deadlines on projects you can't see won't appear.

Exports & AI

Per-project exports

  • CSV — /tasks/projects/<slug>/export.csv. Task list with status, priority, assignee, dates.
  • PDF — /tasks/projects/<slug>/export.pdf. Status report, branded, suitable for stakeholder distribution.

AI summary

From a project page, click Summarize. Helm calls Claude to produce a short narrative covering current status, recent activity, blockers, and upcoming deadlines.

Summaries are cached. The first request returns the cached version; click Refresh to regenerate. Each summary is stamped with the model version and the time it was generated.

Project Online (PWA) import

Staff-only wizard at /tasks/import/project-online/ for ingesting projects from a Microsoft Project Online instance into Helm. Maps PWA fields to Helm projects + tasks; preserves status and assignees where matchable.

Scheduled Jobs & Health Metrics

Scheduled jobs dashboard

Staff-only at /scheduling/. One row per scheduled command in this service, with the cron expression (display only — the scheduler is external), last run, status, recent-24-runs sparkline, and a per-job detail page showing the last 100 runs.

Helm's registered jobs:

  • helm-notify-due-tasks — daily, 09:00 UTC. Fires task_due_soon and task_overdue. Idempotent.

The enabled flag is display-only — toggling it does not pause the underlying cron.

Operational metrics

GET /api/v1/metrics/ returns JSON counters. Auth: staff session OR Authorization: Bearer $HELM_METRICS_TOKEN (for external pollers).

The four canary booleans under flags:

FlagWhat flipping true means
audit_silent_24hNo AuditLog rows written in 24 hours. The compliance trail has gone silent.
cron_silent_24hNo CommandRun rows in 24 hours — the scheduler isn't firing.
cron_failures_24hAt least one scheduled job errored in the last 24 hours.
notifications_failingAt least one delivery failed.

healthy: false (top-level, not nested under flags) is the simplest alert condition. Polled every 15min by GitHub Actions; flags also render inline as green/red chips in the staff "Ops canaries" sub-card on the dashboard.
